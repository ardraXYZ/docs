## Execution flow

1. **Authentication:** Pilots open ArdraHubbot, sign in via SIWS or NextAuth, and link Solana/EVM wallets exactly once.
2. **Preset selection:** Each DEX ships with an approved parameter pack (pairs, starting size, loss limits, funding alerts).
3. **Orchestration:** The backend spins the bot up inside an isolated worker, injects encrypted credentials, and turns on the log stream.
4. **Telemetry:** Fills, funding updates, errors, and stops push to the dashboard and Telegram in real time.
5. **Shutdown:** When the pilot pauses or the bot detects risk, the worker finalizes, persists the last state, and frees the slot.

## Inputs the control plane expects

- **Perp venue:** Aster, Hyperliquid, Backpack, Avantis, Pacifica, ApeX, and the rest of the supported orbit.
- **Wallets and API keys:** Never stored in extensions; Prisma keeps them encrypted at rest and decrypts in-memory per request.
- **Presets:** Define schedule, symbols, and farming intent (volume, maker rebate, funding). Squads can clone them for partner playbooks.

## Best practices

- Use only presets vetted by the Ardra team or reviewed by your squad.
- Keep ArdraHubbot open during long sessions so you can answer confirmation prompts.
- Run weekly reviews by exporting logs to reconcile actual versus expected funding and fees.
